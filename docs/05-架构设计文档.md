### 1.2 架构原则

| 原则 | 说明 |
|------|------|
| 独立运行 | 每个模块无跨模块依赖，可单独学习 |
| 编号即顺序 | 目录/文件编号直接映射学习顺序 |
| 渐进深入 | 从概念→算法→评估→应用的递进结构 |
| 双线并行 | 每个算法先手动实现，再对比sklearn |
