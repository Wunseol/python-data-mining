## 1. 测试策略

### 1.1 测试层级

| 层级 | 方法 | 说明 |
|------|------|------|
| 语法检查 | `python -m py_compile` | 每个文件可编译无错误 |
| 单元测试 | `pytest tests/ -v` | 自动化测试核心算法函数 |
| Lint 检查 | `flake8 --max-line-length=120 --exclude=.git,__pycache__,venv,docs .` | 代码风格检查 |
| 类型检查 | `mypy --ignore-missing-imports --no-strict-optional .` | 静态类型检查 |
| 运行测试 | 手动执行每个 .py | 输出无异常 |
| 结果验证 | 检查关键输出值 | 手动实现 vs sklearn 结果对齐 |

### 1.2 测试原则

- 每个模块可独立运行，无需外部数据或服务
- 手动实现的核心算法需与 sklearn 结果在合理误差内一致
- 可视化输出需人工检查图表正确性
- 随机种子固定 `random_state=42` 或 `np.random.seed(42)`，确保结果可复现
- 测试文件使用 `importlib` 动态导入源码模块，避免跨目录依赖
