# 项目文档重构与优化 Spec

## Why
项目经过多轮迭代（内容补全、新技术添加、代码修复），现有文档严重滞后：源码链接失效、统计数据过时（55文件→78文件、39算法→115+、37测试→119）、缺少7个新增模块的文档、无统一导航、软件工程图缺失。需要全面重构文档，使其与当前源码深度结合，让学习者能跟随一条清晰路线从零学完整个项目。

## What Changes
- 重构文档序号和结构：学习路线在前（01-08），项目文档在后（09-12）
- 每篇文档开头添加导航栏，前后文档可跳转
- 所有源码链接适配 GitHub（`blob/main/` 相对路径）和 VSCode（`file:///` 绝对路径）
- 更新所有统计数据至当前值
- 补充7个新增模块的文档内容
- 用 Mermaid 绘制软件工程图：用例图、活动图、组件图、类图、数据流图、状态图、部署图、序列图
- 设计一条完整学习路线，从快速上手到毕业产出
- 项目文档（需求、架构、开发、测试、报告）排在学习路线之后

## Impact
- Affected code: docs/ 目录下所有 .md 文件将被重写
- Affected docs: README.md 需同步更新
- 新增文件：无（仅重写现有文件）

## ADDED Requirements

### Requirement: 文档导航系统
每篇文档开头 SHALL 包含导航栏，格式为：
```
> 🏠 [项目首页](../readme.md) | 📚 [文档中心](./README.md) | ⬅ [上一篇](./XX.md) | 📍 当前文档 | ➡ [下一篇](./XX.md)
```

### Requirement: 源码深度链接
所有源码引用 SHALL 同时提供 GitHub 链接和 VSCode 链接：
- GitHub: `[文件名](../XX_模块/子目录/文件名.py#L行号)`
- VSCode: 链接格式 `[文件名](file:///absolute/path/to/file.py#L行号)`
- 关键函数 SHALL 标注行号

### Requirement: Mermaid 软件工程图
文档 SHALL 包含以下 Mermaid 图，且所有图 SHALL 基于当前项目实际结构：
1. **用例图**（需求规格）：学习者、贡献者与系统的交互
2. **活动图**（学习路线）：CRISP-DM 流程对应的学习活动
3. **组件图**（架构设计）：10个模块的组件关系
4. **类图**（详细设计）：核心算法的类结构
5. **数据流图**（详细设计）：数据从输入到输出的流转
6. **状态图**（详细设计）：模型训练状态转换
7. **部署图**（架构设计）：本地环境部署
8. **序列图**（实施文档）：模块运行时的调用序列

### Requirement: 学习路线设计
文档 SHALL 设计一条完整学习路线：
- 第0步：快速上手（环境搭建）
- 第1-4阶段：对应4个学习阶段
- 每个模块包含：运行命令、关注要点、动手练习、关键源码跳转
- 阶段自检清单
- 毕业产出

### Requirement: 项目文档排后
需求规格、架构设计、详细设计、实施开发、测试质量、项目报告等贡献者文档 SHALL 排在学习路线之后，编号 09-14。
