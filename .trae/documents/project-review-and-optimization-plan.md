# 项目阅读与优化准备计划

## 项目概况

**项目名称**：Python 数据挖掘学习路径（python-data-mining）

**项目定位**：系统化的数据挖掘知识库与代码实现集，遵循 CRISP-DM 标准流程，按"认知与数据 → 预测建模 → 模式发现 → 场景实战"4阶段渐进学习。

**项目规模**：
- 顶级模块数：10（00-09）
- Python 源文件数：55
- 知识方向数：4阶段 / 10模块 / 30+子方向
- 涵盖算法：39+

## 代码风格与约定

1. **模块结构模板**：每个 .py 文件遵循统一模板（docstring → import → 分节函数 → `if __name__` 主程序）
2. **分节标记**：使用 `# ====...====` 分隔各功能节
3. **中文注释**：关键算法含详细中文注释和直觉解释
4. **双线并行**：核心算法先手动实现，再对比 sklearn
5. **独立运行**：每个模块无跨模块依赖，可单独 `python xxx.py` 运行
6. **自包含数据**：大部分模块使用 sklearn 内置数据集或合成数据，少数依赖外部数据文件
7. **中文字体**：统一使用 `plt.rcParams['font.sans-serif'] = ['SimHei', 'Microsoft YaHei']`
8. **随机种子**：固定 `random_state=42` 或 `np.random.seed(42)`
9. **命名规范**：函数/变量 snake_case，类 PascalCase，目录 `编号_中文名称`

## 已识别的优化方向

基于项目文档和源码阅读，以下为已识别的潜在优化方向（按优先级排列）：

### P0 - 代码质量与可维护性

1. **缺乏自动化测试**：当前仅有手动运行验证（docs/08-测试与质量文档.md），无 pytest 单元测试，requirements.txt 中 pytest 已注释
2. **缺乏 lint/typecheck**：无 flake8/mypy 配置，requirements.txt 中已注释但未启用
3. **外部数据依赖**：04_分类算法/02_朴素贝叶斯/email/ 和 04_分类算法/04_支持向量机/testDigits/ 依赖外部数据文件，违反 NFR2（自包含数据）
4. **重复的样板代码**：每个模块都重复 `plt.rcParams` 设置，可抽取为公共工具模块

### P1 - 代码健壮性

5. **异常处理不足**：核心算法函数缺少输入验证和异常处理（如距离度量函数未检查向量维度一致性）
6. **硬编码问题**：部分模块存在硬编码路径（如文件保存路径 `plt.savefig('随机森林特征重要性.png')`）
7. **类型提示缺失**：函数签名缺少 type hints，不利于 IDE 提示和静态检查

### P2 - 功能与体验

8. **缺少练习题**：项目报告（docs/09）已标记为 P1 改进方向
9. **缺少 Jupyter Notebook**：项目报告已标记为 P2 改进方向
10. **可视化输出管理**：`plt.show()` 在无 GUI 环境下会报错，应增加非交互后端支持

### P3 - 扩展性

11. **缺少分布式计算**：项目报告已标记为 P3 改进方向（Spark/PySpark）
12. **深度学习模块覆盖有限**：仅有感知机/MLP/CNN，缺少 RNN/LSTM/Transformer 等
